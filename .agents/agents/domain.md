# Domain Documentation

The engineering skills read these files to understand the project's domain language and architectural history.

## Layout: Single-Context

- **Glossary**: `CONTEXT.md` at the repo root.
- **Decisions**: `docs/adr/` directory at the repo root.

## Consumer Rules

- **Terminology**: Skills like `improve-codebase-architecture` and `diagnose` must use terms defined in `CONTEXT.md`.
- **History**: Skills like `tdd` should consult `docs/adr/` to avoid revisiting resolved trade-offs.
